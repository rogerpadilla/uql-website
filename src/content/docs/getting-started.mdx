**[UQL](/)** is the JSON-native query protocol for TypeScript. It is engineered to be **fast**, **safe**, and **universally compatible**.

- **Runs Everywhere**: Node.js, Bun, Deno, Cloudflare Workers, Electron, React Native, and the Browser.
- **Unified API**: A consistent interface for PostgreSQL, MySQL, MariaDB, SQLite, LibSQL, Neon, D1, and MongoDB.

:::note[New to UQL? Start here]

For your first week with UQL, follow this path:

1. **[Quick Start](/getting-started)** (this page) — install, configure, and run your first query.  
2. **[Entities](/entities/basic)** — define your models and types.  
3. **[Relations](/entities/relations)** — map associations between entities.  
4. **[Querying](/querying/querier)** — learn `$select`, `$where`, `$sort`, and friends.  
5. **[Transactions](/querying/transactions)** — wrap operations safely.  
6. **[Migrations](/migrations)** — keep your database in sync with your code.

:::

Install the core and your preferred driver:

  <TabItem label="npm">

    ```sh
    npm install uql-orm pg # or mysql2, better-sqlite3, mongodb, etc.
    ```
  </TabItem>
  <TabItem label="bun">
    ```sh
    bun add uql-orm pg # or mysql2, better-sqlite3, mongodb, etc.
    ```
  </TabItem>
  <TabItem label="pnpm">
    ```sh
    pnpm add uql-orm pg # or mysql2, better-sqlite3, mongodb, etc.
    ```
  </TabItem>
</Tabs>

:::tip[TypeScript Setup]
UQL uses decorators. Make sure your `tsconfig.json` has:

```json
{
  "compilerOptions": {
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```

UQL is Modern Pure ESM — ensure your project's `module` supports ESM imports (e.g., `NodeNext`, `ESNext`, `Bundler`).
:::

## 2. Fast-Track Example

Here is a complete example of defining an entity, setting up a pool, and running a query.

```ts
// entities.ts
import { Entity, Id, Field } from 'uql-orm';

@Entity()
export class User {
  @Id({ type: 'uuid' })
  id?: string;

  @Field({ unique: true })
  email?: string;

  @Field()
  name?: string;
}

// uql.config.ts
import { PgQuerierPool } from 'uql-orm/postgres';
import { User } from './entities.js';

export const pool = new PgQuerierPool(
  { host: 'localhost', database: 'uql_app' },
  { entities: [User] }
);

// app.ts
import { pool } from './uql.config.js';
import { User } from './entities.js';

const users = await pool.transaction(async (querier) => {
  return await querier.findMany(User, {
    $select: { id: true, name: true },
    $where: { email: { $endsWith: '@uql-orm.dev' } },
    $limit: 10,
  });
});

console.log(users);
```

## Import Paths

UQL uses subpath exports to keep your bundles lean:

| Path | Purpose |
| :--- | :--- |
| `uql-orm` | Core decorators, types, and query utilities |
| `uql-orm/postgres` | PostgreSQL driver pool |
| `uql-orm/cockroach` | CockroachDB driver pool |
| `uql-orm/mysql` | MySQL driver pool (also works with MariaDB) |
| `uql-orm/maria` | MariaDB driver pool (dedicated `mariadb` driver) |
| `uql-orm/sqlite` | SQLite driver pool |
| `uql-orm/libsql` | LibSQL / Turso driver pool |
| `uql-orm/neon` | Neon serverless driver pool |
| `uql-orm/d1` | Cloudflare D1 driver pool |
| `uql-orm/mongo` | MongoDB driver pool |
| `uql-orm/express` | Express middleware (auto-generated REST APIs) |
| `uql-orm/browser` | Browser HTTP querier |
| `uql-orm/migrate` | Migration engine & CLI |

## Common pitfalls

::::tip
- **Decorators not working**: Ensure `experimentalDecorators` and `emitDecoratorMetadata` are enabled in `tsconfig.json`, and that your bundler supports decorators.  
- **ESM import issues**: UQL is Pure ESM — set your `module` to `NodeNext`, `ESNext`, or `Bundler`, and avoid mixing `require()` with `import`.  
- **Connection errors**: Double‑check database credentials and that your driver package (for example `pg`, `mysql2`, or `better-sqlite3`) is installed and compatible with your runtime.  
::::
