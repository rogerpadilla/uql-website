## Overview

This guide shows how to use **UQL** with **Next.js App Router** to build a type-safe API and server actions.

You will:

- Configure UQL in a shared module.
- Use it from **route handlers**.
- Optionally, call it from **server actions**.

---

## 1. Install dependencies

```sh
npm install uql-orm pg server-only
```

---

## 2. Define entities and UQL pool

UQL's decorators need two flags that `create-next-app` does not include; add them to `tsconfig.json` (both Turbopack and Webpack support them):

```json
{
  "compilerOptions": {
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```

Create an `entities.ts` file in your app (for example in `src/db/entities.ts`):

```ts
import { Entity, Id, Field } from 'uql-orm';

@Entity()
export class User {
  @Id({ type: 'uuid' })
  id?: string;

  @Field({ unique: true })
  email?: string;

  @Field()
  name?: string;
}
```

Then create a UQL pool in `src/db/uql.ts`:

```ts
import 'server-only';
import { setQuerierPool } from 'uql-orm';
import { PgQuerierPool } from 'uql-orm/postgres';
import './entities'; // importing the module registers the decorated entities

// reuse one pool across `next dev` hot reloads; a fresh pool per reload leaks connections
declare global {
  var uqlPool: PgQuerierPool | undefined;
}

export const uqlPool = (globalThis.uqlPool ??= new PgQuerierPool({
  host: process.env.POSTGRES_HOST,
  database: process.env.POSTGRES_DB,
  user: process.env.POSTGRES_USER,
  password: process.env.POSTGRES_PASSWORD,
}));

// the default pool: what createFetchHandler and getQuerier() use
setQuerierPool(uqlPool);
```

The `server-only` import turns any accidental import from client code into a clear build-time error, and the `globalThis` singleton is the same pattern Prisma and Drizzle recommend for dev-server hot reloading: on re-evaluation the module reuses the pool stored on `globalThis` instead of creating a new one.

:::tip[Vercel]
On Vercel's Fluid compute, register the inner `pg` pool so idle connections close before the function suspends: `attachDatabasePool(uqlPool.pool)` from `@vercel/functions`. Keep the default Node.js runtime; `pg` is not edge-compatible.
:::

## 3. Use UQL in route handlers

Create an API route such as `app/api/users/route.ts`:

```ts
import { NextResponse } from 'next/server';
import { uqlPool } from '@/db/uql';
import { User } from '@/db/entities';

export async function GET() {
  const users = await uqlPool.withQuerier((querier) =>
    querier.findMany(User, {
      $select: { id: true, name: true, email: true },
      $limit: 20,
    }),
  );

  return NextResponse.json(users);
}
```

This keeps your query fully type-safe and works across all supported databases.

### Auto-generated CRUD with a catch-all route

Instead of hand-writing one route per entity, mount the [HTTP transport core](/extensions-http) in a single catch-all route handler, `app/api/uql/[[...uql]]/route.ts`:

```ts
import { createFetchHandler } from 'uql-orm/http';
import '@/db/uql'; // creates the pool and sets it as the default
import { User } from '@/db/entities';

const handler = createFetchHandler({
  include: [User],
  basePath: '/api/uql',
});

export { handler as GET, handler as HEAD, handler as POST, handler as PUT, handler as PATCH, handler as DELETE };
```

Every included entity now has typed REST endpoints (`/api/uql/user`, ...) consumable from the browser with [`HttpQuerier`](/extensions-browser). Handle auth and tenant scoping server-side with hooks, or - for real multi-tenancy - pass `getContext` to `createFetchHandler` and add a `security` [filter](/querying/filters), so every request is scoped automatically and can't be bypassed from the client. See [Multi-tenancy](/multi-tenancy).

## 4. Use UQL from server actions

You can also use UQL directly inside **server actions** for form submissions or mutations:

```ts
'use server';

import { revalidatePath } from 'next/cache';
import { uqlPool } from '@/db/uql';
import { User } from '@/db/entities';

export async function createUser(formData: FormData) {
  const email = formData.get('email') as string;
  const name = formData.get('name') as string;

  await uqlPool.transaction(async (querier) => {
    await querier.insertOne(User, { email, name });
  });

  revalidatePath('/users'); // refresh pages that render this data
}
```

## 5. Add AI & semantic search

Once you have entities in place, you can plug in **semantic search** for AI features. Add an entity with a vector field to `src/db/entities.ts`:

```ts
import { Entity, Id, Field } from 'uql-orm';

@Entity()
export class Article {
  @Id({ type: 'uuid' })
  id?: string;

  @Field()
  title?: string;

  @Field({ type: 'vector', dimensions: 1536 })
  embedding?: number[];
}
```

Then query it by similarity:

```ts
import { uqlPool } from '@/db/uql';
import { Article } from '@/db/entities';

export async function searchArticles(queryEmbedding: number[]) {
  return uqlPool.withQuerier((querier) =>
    querier.findMany(Article, {
      $select: { id: true, title: true },
      $sort: { embedding: { $vector: queryEmbedding, $distance: 'cosine' } },
      $limit: 10,
    }),
  );
}
```

See the [AI & RAG](/ai-semantic-search) page for end-to-end patterns.
